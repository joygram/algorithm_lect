01. 자료란 뭘까요?

자료는 컴퓨터가 다루는 모든 정보예요. 숫자, 문자, 사진, 동영상까지 다 자료입니다.

자료 추상화는 "어떤 걸 넣고 빼고 찾을 수 있다"만 정해 두고, "어디에 어떻게 저장하는지"는 숨기는 거예요. 시험에 ADT = Abstract Data Type(추상 자료형, 에이디티·앱스트랙트 데이터 타입)이라고 나옵니다.

  • 데이터 타입: 정수(1, 2, 3), 실수(1.5), 문자(‘가’), 문자열("안녕"), 참/거짓 등
  • 추상화: 넣기, 빼기, 찾기 같은 동작만 정하고, 저장 방식은 나중에 정함

02. 자료는 어떻게 저장될까?

컴퓨터 안에서는 모든 게 0과 1(2진수)로 바뀌어 저장돼요.

  • 비트(bit): 0 하나 또는 1 하나. Binary + digit의 줄임말.
  • 바이트(byte): 비트 8개 = 1바이트. 글자 하나는 보통 1바이트 이상 사용.

02-1. 진법이란?

진법은 수를 셀 때 "몇 개씩 묶어서 자릿수를 올리느냐"를 정한 거예요. 우리가 쓰는 일상 숫자는 10진법(0~9, 10이 되면 다음 자리), 컴퓨터는 2진법(0과 1만), 프로그래밍에서는 8진법, 16진법도 자주 씁니다.

  • 2진수: 0, 1만 사용. 예: 1010₂ = 1×2³ + 0×2² + 1×2¹ + 0×2⁰ = 10
  • 8진수: 0~7. 예: 12₈ = 1×8¹ + 2×8⁰ = 10
  • 10진수: 0~9. 우리가 쓰는 수.
  • 16진수: 0~9, A(10), B(11), C(12), D(13), E(14), F(15). 예: 1A₁₆ = 1×16 + 10 = 26

02-2. 진법 변환 (중요 · 시험 단골)

① 10진수 → 2진수: 10진수를 2로 계속 나눠서 나머지를 아래에서 위로 읽으면 됩니다.

예: 13을 2진수로? 13÷2=6…1, 6÷2=3…0, 3÷2=1…1, 1÷2=0…1 → 나머지 순서: 1,0,1,1 → 거꾸로 읽으면 1101₂ = 13

② 2진수 → 10진수: 각 자리에 2의 거듭제곱을 곱해서 더합니다.

예: 1101₂ = 1×2³ + 1×2² + 0×2¹ + 1×2⁰ = 8+4+0+1 = 13

③ 10진수 → 16진수: 10진수를 16으로 계속 나눈 나머지를 모아요. 10~15는 A~F로 씁니다.

예: 255 → 255÷16=15…15 → 15÷16=0…15 → 나머지 15,15 → FF₁₆ = 255

④ 2진수 ↔ 16진수: 2진수 네 자리씩 묶으면 16진수 한 자리예요. 0000=0, 0001=1, … 1111=F.

예: 1111 1010₂ → 1111=F, 1010=A → FA₁₆. 반대로 FA₁₆ → F=1111, A=1010 → 11111010₂

02-3. 보수(補數) 개념 (시험 단골)

컴퓨터에서 음수를 표현하거나 뺄셈을 할 때 "보수"를 씁니다. 어떤 수에 더했을 때 자릿수가 한 단계 올라가게 만드는 수예요.

① 1의 보수: 2진수에서 0은 1로, 1은 0으로 바꾼 거예요. "비트를 반전"한다고 해요.

예: 0011 (3)의 1의 보수 → 1100. 0011 + 1100 = 1111 (모두 1). 4비트에서 1111+1 = 10000이 되면 자리 올림이 나요.

② 2의 보수: 1의 보수에 1을 더한 값이에요. 컴퓨터는 보통 2의 보수로 음수를 표현해요.

예: 0011 (3)의 2의 보수 → 1의 보수 1100 + 1 = 1101. 이 1101을 "음수 -3"으로 쓰는 방식이에요. 0011 + 1101 = 10000 → 4비트만 쓰면 0000이 되니까 3 + (-3) = 0이 됩니다.

정리: n비트일 때, 어떤 수의 2의 보수 = 그 수의 1의 보수 + 1. 2의 보수로 표현한 수는 "음수"로 해석합니다. 시험에 "8비트에서 -5의 2의 보수 표현" 같은 문제가 나와요. (5 = 0000 0101 → 1의 보수 1111 1010 → 2의 보수 1111 1011)

02-4. 문자 인코딩

글자(문자)도 숫자로 바꿔서 저장해요. 그 규칙이 인코딩이에요.

  • ASCII: 영문·숫자·기호 등 128개(7비트). 예: 'A' = 65.
  • 유니코드(Unicode): 전 세계 문자를 하나의 번호로 매긴 표. 한글, 이모지까지 포함.
  • UTF-8: 유니코드를 실제로 저장할 때 쓰는 방식 중 하나. 1바이트~4바이트로 한 글자를 저장.

03. 자료구조와 알고리즘

자료구조 = 자료를 어떤 순서·형태로 담아 둘지 정한 구조 (줄로 세우기, 쌓아 두기 등).

알고리즘 = 문제를 풀기 위한 단계별 방법. 좋은 자료구조를 고르면 알고리즘이 더 빨라져요.

04. 리스트(List) — 순서 있는 나열

리스트는 순서가 있는 자료의 나열이에요. 같은 종류의 값들이 한 줄로 배치됩니다.

  • 배열: 칸이 연달아 있어서 "몇 번째 칸"으로 바로 찾기 좋아요. 삽입·삭제 시 다른 칸을 밀어야 할 수 있어요.
  • 연결 리스트: 각 노드가 "다음 노드 주소"를 갖고 있어서, 가운데 삽입·삭제가 배열보다 유리해요. 머리(head), 꼬리(tail) 용어가 시험에 나옵니다.

실습: 진법·보수 실습: 리스트 브라우저에서 파이썬 실습 기출 유형 예제
