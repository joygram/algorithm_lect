II단원 확장 — 게임 시스템 실습과제

학습한 스택·큐를 메뉴 히스토리·대사 큐로 확장하기 (추가 강의)

← 전체 목차 II단원 목차

이 페이지는 강의 외 추가 강의예요. II단원에서 배운 스택·큐를 게임의 메뉴 뒤로가기, 대사·이벤트 순서라는 구체적인 시스템으로 확장합니다. 각 주제마다 자료구조 복기 → 사용 API → 실제 구현 활용 순으로 짚어요.

1. 스택 — 메뉴 화면 히스토리

게임에서 타이틀 → 메인 → 설정처럼 들어갈 때마다 화면이 쌓이고, "뒤로"를 누르면 가장 최근 화면으로 돌아갑니다. 이게 스택(LIFO)이에요.

📌 복기: 스택

  • 연산: push(맨 위에 넣기), pop(맨 위에서 꺼내기). 나중에 넣은 게 먼저 나옴 = LIFO.
  • 메뉴 "들어가기" = push(현재 화면 이름), "뒤로" = pop 후 이전 화면 표시.

📌 사용하는 파이썬 API

리스트.append(값)
맨 뒤에 넣기 → 스택의 push. 리스트의 "맨 뒤"를 스택의 "맨 위(top)"로 봅니다.
리스트.pop()
괄호 비우면 맨 뒤에서 하나 꺼내 반환 → 스택의 pop.

🎮 실제 구현에서 쓰는 곳

  • 메뉴 스택: "타이틀" → "메인" → "설정" 들어갈 때마다 stack.append("설정")처럼 화면 이름을 쌓고, "뒤로" 시 stack.pop()으로 이전 화면 이름을 꺼내서 그 화면을 다시 그립니다.
  • 언더테일처럼 메뉴가 겹겹이 쌓이는 구조가 스택이에요.

👇 아래 코드가 실습창과 동일한 스택 예제예요. push = append, pop = pop().

# 스택: 메뉴 히스토리 흉내
stack = []
stack.append("타이틀")
stack.append("메인")
stack.append("설정")
print("들어간 순서:", stack)
# 뒤로 가기
screen = stack.pop()
print("뒤로 →", screen, ", 남은 스택:", stack)
🐍 실습: 스택 (메뉴)

2. 큐 — 대사·이벤트 순서

대사 한 줄이 나온 순서대로 화면에 띄워지고, "다음"을 누르면 다음 대사가 나옵니다. 먼저 넣은 게 먼저 나오는 FIFO가 큐예요.

📌 복기: 큐

  • 연산: enqueue(맨 뒤에 넣기), dequeue(맨 앞에서 꺼내기). 먼저 넣은 게 먼저 나옴 = FIFO.
  • 대사 한 줄 추가 = enqueue, "다음 대사 표시" = dequeue로 한 줄 꺼내서 화면에 출력.

📌 사용하는 파이썬 API

from collections import deque
덱(double-ended queue): 앞·뒤 모두 O(1)로 넣고 뺄 수 있어요. 큐로 쓸 때는 "뒤에 넣고 앞에서 뺌".
덱.append(값), 덱.popleft()
append = enqueue, popleft() = dequeue. list의 pop(0)은 비용이 크므로 큐에는 deque를 씁니다.

🎮 실제 구현에서 쓰는 곳

  • 대사 큐: 이벤트에서 나올 대사 줄들을 순서대로 q.append("안녕!"), q.append("선택지가 있어요")로 넣고, 플레이어가 "다음"을 누를 때마다 line = q.popleft()로 한 줄 꺼내서 화면에 표시.
  • 선택지 전에 쌓인 대사가 순서대로 재생되는 구조가 큐예요.

👇 아래 코드가 실습창과 동일한 큐 예제예요. 맨 앞에서 꺼낼 때 popleft().

# 큐: 대사 순서 흉내
from collections import deque
q = deque(["안녕!", "선택지가 있어요", "1. 예 2. 아니오"])
print("대사 큐:", list(q))
out = q.popleft()
print("다음 대사 →", out)
print("남은 큐:", list(q))
🐍 실습: 큐 (대사)
