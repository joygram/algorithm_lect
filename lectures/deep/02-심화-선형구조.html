II. 선형 구조 — 심화 실습

파이썬으로 메뉴 뒤로가기·대사 큐 구현 (4단계)

← II단원 목차 전체 목차

스택과 큐를 파이썬으로 써서 메뉴 화면 히스토리, 대사 순서를 구현해 보는 심화 실습이에요.

🎮 이 심화에서 다루는 게임 활용

스택 = 타이틀 → 메인 → 설정 들어갈 때마다 push, "뒤로" 누르면 pop. (LIFO) 메뉴뿐 아니라 씬 전환 히스토리, "이전 대화로 돌아가기"에도 쓸 수 있어요. 큐 = 대사 줄을 순서대로 enqueue, "다음" 누를 때마다 dequeue. (FIFO) 이벤트·대사·메시지가 나온 순서대로 처리될 때 모두 큐로 다룹니다.

심화 실습 4단계

  1. 1단계: 스택으로 메뉴 화면 이름 쌓기 — 개념 스택은 맨 위에만 넣고 빼는 구조(LIFO). 활용 리스트에 "타이틀", "메인", "설정"을 append로 넣고, pop으로 "뒤로" 할 때마다 꺼낸 값이 "현재 화면"이에요.
  2. 2단계: 큐로 대사 줄 넣고 꺼내기 — 개념 큐는 맨 뒤에 넣고 맨 앞에서 빼는 구조(FIFO). 활용 deque에 "안녕!", "선택지가 있어요"를 넣고 popleft()로 한 줄씩 꺼내서 화면에 표시하는 흐름이에요.
  3. 3단계: 스택+큐 함께 쓰기 — 메뉴 스택과 대사 큐를 각각 변수로 두고, "메뉴 들어가기" / "대사 다음" 시뮬레이션을 print로 확인. 게임에서는 한 화면에 메뉴 계층(스택)과 대사 줄(큐)이 동시에 쓰여요.
  4. 4단계: 간단한 메뉴 루프 흉내 — 미리 정해 둔 push/pop 순서를 코드로 써서 "들어가기·뒤로가기"가 스택에 어떻게 쌓이는지 확인. 디버깅할 때 "지금 스택에 뭐가 쌓여 있나?"를 보면 메뉴 흐름을 추적할 수 있어요.

1·2단계: 스택·큐 실습

개념 복기 스택 = LIFO(나중에 넣은 게 먼저 나옴) → 메뉴 "뒤로". 큐 = FIFO(먼저 넣은 게 먼저 나옴) → 대사·이벤트 순서. 파이썬에서는 리스트로 스택(append/pop), collections.deque로 큐(append/popleft)를 씁니다.

3·4단계

활용 실제 게임에서는 "현재 화면" = 스택 top, "다음에 나올 대사" = 큐 front. 같은 페이지의 통합 파이썬 실습에서 스택 예제와 큐 예제를 번갈아 불러와서, "메뉴 이름 push/pop", "대사 popleft"를 한 번씩 실행해 보세요.
